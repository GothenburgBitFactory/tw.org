Work in Progress

This design document is a work in progress, and subject to change. Once finalized, the feature will be scheduled for an upcoming release.

List Review

To support GTD-methodology users, Taskwarrior must support the notion of a review. This is a periodic process that aims to align a task list with reality. This involves looking at tasks, ensuring the information is current and correct, and that the task is still relevant, with the right metadata.

You don't have to follow the GTD methodology to see that this is a good idea - an inaccurate task list is suboptimal.

This design document is a collaboration between Taskwarrior and Fidel Mato, author of trev, an extension script that implements a review process.

Tasksh - Task Shell

The tasksh project provides an interactive shell for Taskwarrior, and is an ideal platform on which to build review features. It has built-in libreadline support.

Minimal Features

A minimal review feature (a good place to start) will need to satisfy the basic needs of a review process, and there are always users that do not want elaborate features. Here is a list of the minimal set of features:

  • Ability to start and end a review session, where zero or more tasks are reviewed.
  • During review, allow modification to tasks.
  • Remember which tasks have been reviewed, to allow later resumption.
  • Rotate through tasks on a periodic basis, say weekly or monthly, perhaps using a review UDA timestamp.
  • Simply running the review command, with no arguments or additional setup should walk the user through a properly sequenced review of all tasks that need review.

Extra Features

Trev provides more than a minimal set of features. Taskwarrior will provide additional features, but they will be optional. Here is a list of the potential extra features:

  • Ability to traverse forwards and backwards through the list.
  • Ability to apply a filter to the list being reviewed.
  • Progress indicator.
  • Built-in actions, to make the review easier. This means there could be one-letter commands that, for example, delete, start, wait a task.
  • Track review time spent, and estimated review time remaining.
  • Review could be made context-aware, perhaps indirectly.
