Troubleshooting Sync

This page is updated to reflect the latest information and help available. If there is something you think should be listed here, please tell us.

Taskwarrior

  • Are you using Taskwarrior 2.3.0 or later? If not, upgrade.
  • Did you build Taskwarrior from git? If so, have you updated recently, and are you on the correct branch? Remember, the master branch always represents the most recently released version.
  • Did you build Taskwarrior from source, and if so, which version of GnuTLS did you use?
  • Cert, key installed, readable and not empty? Making sure it is not empty is important.
  • Configuration properly points to cert/key?
  • taskd.credentials correct?
  • Are you using a self-signed cert? If so, did you specify taskd.ca or set taskd.trust=yes?
  • Have you upgraded from a very old version of Taskwarrior ending up having duplicate UUIDs?
  • Set debug.tls=2 in the configuration. You get detailed TLS diagnostics that way.

Taskserver

  • Are you using Taskserver version 1.0.0 or later? If not, upgrade.
  • Did you build Taskserver from git? If so, have you updated recently, and are you on the correct branch? Remember, the master branch always represents the most recently released version.
  • Did you build Taskserver from source, and if so, which version of GnuTLS did you use?
  • If you built from source, did the demo work?
  • Is the server running? Check with ps -leaf | grep taskd.
  • Are there helpful messages in the taskd.log file?
  • Is the client (taskwarrior) able to communicate through your firewall to the server (taskserver)? By default, port 53589 is used, and must be open on the server.
  • If you are unable to open firewall ports, you can use an SSH Tunnel to route port 53589 traffic over port 22.
  • Did you follow the instructions for setup exactly? This is the largest cause of misconfiguration.
  • Did you edit the generate.server script to contain your dns_name and ip_address?
  • Setting debug.tls=2 for both client and server generates diagnostic output that can help diagnose problems with TLS handshake, certs, keys, and ciphers. For Taskserver, with debug.tls=2, run the server interactively rather than as a daemon, to see the diagnostics.
  • Setting ciphers (Taskserver) or taskd.ciphers (Taskwarrior) can force the use of different ciphers. Use gnutls-cli --list to see a list of installed ciphers, and confirm that there is overlap between client and server.
  • In the event of a self-signed cert being used, is the CA cert configured? Does the cert file have contents?
  • Alternatively, is taskd.trust=yes set for Taskwarrior?

Report Problems

Please report problems that you see. When reporting a problem, please include:

  • Output from taskd diagnostics command.
  • Output from task diagnostics command.
  • The taskd.log contents.
  • Impress us and mention the codeword 'Weasel', to prove that you at least read this page all the way to the end.
The preferred channel for reporting issues is on our answers.tasktools.org Q & A site.
